CRITICAL FIXES DETAIL
================================================================================

#1 Hardcoded API Key (CRITICAL BREACH)
   Location: src/lib/fraud-check.ts:62
   Severity: CRITICAL
   Status: FIXED
   
   Issue: Production API key hardcoded in source code
   Fix: Now uses environment variable BD_COURIER_API_KEY
   Impact: Prevents unauthorized API access if code is leaked

#2 Fraud Risk Algorithm Bug (HIGH)
   Location: src/lib/fraud-check.ts:145-148
   Severity: HIGH (Business Logic)
   Status: FIXED
   
   Issue: Old resolved reports permanently flagged customers high risk
   Fix: Time-based decay (3x weight ≤30d, 2x ≤90d, 1x ≤180d, 0.5x >180d)
   Impact: Accurate risk assessment, prevents false positives

#3 Missing Request Timeouts (HIGH)
   Location: src/lib/fraud-check.ts:52, 73, 123
   Severity: HIGH (DoS Risk)
   Status: FIXED
   
   Issue: External API calls had no timeout (infinite hang)
   Fix: AbortController with 10s/15s timeouts
   Impact: Prevents resource exhaustion from hanging requests

#4 Silent Error Failures (HIGH)
   Location: src/lib/fraud-check.ts:47, 70, 82, 108, 116, 133
   Severity: HIGH (Observability)
   Status: FIXED
   
   Issue: Errors logged without context, silent failures
   Fix: Added [FraudCheck] prefix, warning logs, proper returns
   Impact: Easier debugging, better observability

#5 Insufficient Input Validation (MEDIUM)
   Location: src/lib/fraud-check.ts:110-113
   Severity: MEDIUM (Data Integrity)
   Status: FIXED
   
   Issue: Invalid phone numbers caused silent failures
   Fix: Validation with console.warn for invalid input
   Impact: Better data quality, operational awareness
